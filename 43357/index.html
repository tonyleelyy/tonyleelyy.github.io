Windows & iOS 连续互通——我的解决方案
发表于2020-08-08|更新于2020-08-08|教程系统
|字数总计:2.1k|阅读时长:6分钟|阅读量:

在 Apple 生态中，「连续互通」一直是提升体验和效率的重要一环。在 Apple 介绍 使用“连续互通”连接设备 的页面中，可以看到 Apple 将「连续互通」扩展为 接力、通用剪贴板、iPhone 蜂窝网络通话……等等 12 项功能。

其中我认为最常用也最能提升效率的，应该是「接力」「通用剪贴板」「隔空投送」。

而要在 Windows 系统下实现这些功能，我们需要一些技巧和工具。

虽说是 Win & iOS 连续互通，但其实 Android 系统也同样适用大部分功能，本人知识有限，这里就不做讨论了。

本文使用到的工具：Bark、Windows SMB服务、iCloud、Documents App。

为什么要实现「连续互通」

当想到「要向电脑传输文件」「想将电脑上的一句话复制到手机上」「想用电脑打开手机上的网页」，不少人都会选择伟大的国民社交 App——微信。但就算抛开微信的种种对文件的限制不说，光是在电脑上每次都需要登微信，就已经足够烦人的了。这里容我先谈谈实现「连续互通」的好处：

提升效率，不止一点半点

  • 如果你需要将一个网页链接发送到手机，

    使用微信将需要 11 步：点开微信客户端，点击登陆，在手机确认登陆，复制网址，点击微信，点选文件传输助手或自己的聊天框，粘贴，发送，在手机端复制，在手机端打开浏览器，点击「粘贴并前往」。

    而使用 Bark 只需 4 步：在 Chrome 右键链接，点击 Bark 菜单发送，在手机端打开浏览器，点击「粘贴并前往」。

  • 如果你需要把手机上拍的一条2分钟的1080P视频发送到电脑，

    使用微信将会自动“帮”你把视频压缩成 640x368 30帧的指甲盖大小模式，且需要额外等待微信压缩视频的时间。

    使用 iCloud 同步，上传和下载速度都会极其感人。

    而使用 Windows SMB 服务，实现无损传输的同时，速度几乎可以跑满路由器带宽（受限于 Wi-Fi 信号强度）

保持创作的「流畅感」

创作灵感可不会一直都能保持在脑海中。「连续互通」可以让你实现跨设备无缝工作，我相信这样创作无论如何都比拼凑灵感碎片要来得高效，创作出的内容也会相对高质量。

怎样实现「连续互通」

实际上 iPadOS 也同样适用，为了方便这里统称为「iOS」。

接力 & 通用剪贴板

Mac 和 iOS 设备之间可以共享剪贴板，即实现「在电脑复制，在手机粘贴」或「在手机复制，在电脑粘贴」。

而在 Mac 设备和 iOS 设备上用浏览器打开网页或支持接力的 App，在另一台设备上也能同时看到。

在 Windows 系统下，我们可以实现一半的功能，即「在电脑复制，在手机粘贴」和「网页接力」。

这里使用到 Bark 这个神器。

Bark is an iOS App which allows you to push costumed notifications to your iPhone.

Bark 可以在电脑端向 iOS 设备推送通知，就跟短信，微信的消息通知一样。由于走的是 Apple 的推送服务，所以几乎不会出现延迟。

安装 Bark 的 Chrome 扩展 自动复制版。

Chrome 商店地址，这个版本向 iOS 设备推送消息之后，手机会自动复制消息的内容，无需打开 App，也无需手动操作。

在手机端安装 Bark App

App Store 链接，免费开源无内购。

安装打开后复制 App 内的链接

到刚才安装的 Chrome 插件选项卡中填写。

Chrome 插件内的设置

然后就可以直接使用了。在浏览器中选中文字信息，右键点击 Bark 菜单即可发送。

手机端收到消息后会自动复制，不用手动操作，间接实现了通用剪贴板和网页接力功能。

如果你想更安全一点，也可以浏览 Finb 的 GitHub 项目，将 Bark 服务部署到你的服务器。

隔空投送

不同的 Apple 设备之间可以用「隔空投送」功能快速方便地无线传输文件。

而在 Windows 系统与 iOS 设备之间，我们也可以通过一些技术实现这一功能，同样运用 Wi-Fi 快速地传输文件。

这里使用到的是「Windows SMB」服务，iOS 上自带的「文件」和 App Store 上的「Documents」App。

在控制面板启用文件共享

首先需要打开 控制面板 - 网络和共享中心 - 高级共享设置

将带有（当前配置文件）一项的 网络发现 和 文件和打印机共享 启用。

选择电脑上负责接收文件的文件夹

比如我这里选择桌面，右键 - 属性 - 共享。

点击 高级共享，勾选 共享此文件夹。

额外的安全操作

点击 高级共享 里面的 权限，将自带的 Everyone 删除。

点击 添加 - 高级 - 立即查找。在搜索结果里找到自己的 Windows 用户，点击确定。

确定后回到 权限 面板，可以看到我们的用户已经添加到了列表中，记住自己用户名括号后面的内容，比如我的 50555xxx@qq.com，也可以根据自己的需要分配权限。

获取当前电脑的内网 IP

打开自带的 Windows PowerShell 工具，输入 ipconfig。

记住这里出现的「IPv4 地址」，我这里是 192.168.0.7

打开手机「文件」App，连接服务器

在弹出的页面输入服务器「smb://刚才的IP地址」，比如我的是「smb://192.168.0.7」。

连接身份选择「注册用户」，填入你 Windows 账号名称和密码（名称是刚才括号内的内容）。点击下一步即添加完成。

现在你应该可以在文件 App 内打开你的 SMB 服务，并访问你刚才添加的文件夹了。

效果演示

准备工作完成，现在已经可以实现「隔空投送」的操作了。在文件或图片的分享页面中选择 存储到“文件”，再点击 SMB 服务器，存储，文件就飞快地传到了电脑里。

SMB 服务器的其他妙用

在 iOS 和 macOS 端，接力功能还可以实现非常方便的跨设备工作，比如「在电脑写到一半的文章，在手机上继续写」。

利用刚才部署的 SMB 服务，我们也可以在 Windows 和 iOS 设备之间实现这个功能。

跟「文件」App 不一样的是，Documents 自带一个「同步」功能。将文件夹添加同步之后，App 每次都会帮你把文件夹内的文件下载到手机上，而「文件」App 则不会。

这么做使得就算你的电脑需要关机，只要你在关机前打开 Documents，下拉同步一下，文件就保存在了你的手机里，你可以在手机上继续你的工作。

你在手机里完成修改工作之后，打开电脑，在 App 里下拉同步一下，电脑里的文件就会变成你在手机里修改过的版本。间接实现了跨设备工作~

后记

在 Windows 上实现「连续互通」其实还有很多种方法，比如直接使用 Edge 浏览器将 iCloud 网页版上的 App 打包成桌面应用，使用官方的 iCloud 应用，使用 Send Anywhere，Continue on PC 等 App。我这里仅列出我自己使用过，觉得比较接近原生体验的 App。

而 Bark 和 SMB 真的是神器，还可以实现譬如「扫码通知挪车」「多媒体服务器」之类的操作，等大家去挖掘。
